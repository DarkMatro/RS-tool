1.5. Вкладка Fitting

В правой части находится панель с таблицей линий (1), таблицей параметров текущей выбранной линии (2) и окно со статистикой (3)
На графике (4) отображается спектр с исходными данными (data), сумма линий (sum), разница между data и sum (residual) и погрешность моделирования спектра (3 sigma)
Положение и амплитуду линий можно изменять в таблице (2) и на самом графике перетаскиванием.
Кнопка Add - добавить линию
Кнопка Fit - подогнать модель под 1 текущий спектр
Кнопка Batch - подогнать модель под все спектры, результат выводится в таблицу Deconvoluted dataset на 3 вкладке
Кнопка Guess - автоматически создать модель, подходящую под все спектры.
Параметр Interval - если активен то подгоняется модель только в указанном интервале.
Opt.method - метод оптимизации модели. Доступно 15 методов.
Guess method - Average - при автоматическом создании модели анализируется только один усредненный общий спектр. Average groups - анализируются усредненные спектры групп. All - анализируются все спектры указанные в таблице внизу.
Max peak HWHM - ограничение максимальной полуширины линии при автоматическом определении модели (Guess)
N lines method - используется только при автоматическом определении модели (Guess). После определения состава линий автоматическим Guess по каждому спектру формируется список x0. По каждому интервалу количество линий может для разных спектров может быть разным. Выбор метода влияет на количество линий в итоговой модели. Это количество дальше используется в k-means.
Пример для одного интервала из 5 спектров: [5, 5, 6, 6, 7]
Min - будет выбрано минимальное количество линий (5).
Max - максимальное из всех (7).
Mean - 5.8, округляется до 6
Median - 6
Max noise level - используется только при автоматическом определении модели (Guess). Ниже этого значения интенсивности сигнал принимается за шум и новые линии там не ищутся. Если указать слишком низкое значение то алгоритм будет долго искать все мелкие линии и аппроксимировать весь шум.
l_ratio limit - используется только при автоматическом определении модели (Guess) и только для линий типа Split Pseudo Voight, Pseudo Voight. Ограничивает значение параметра l_ratio, чтобы линия оставалась преимущественно Гауссианой с небольшим добавлением линий Лоренца.
Include x0 - используется при Batch Fit, при включенном параметре в датасет будут включены значения x0, а не только амплитуды линий.
 
SPLIT BY INTERVALS - таблица границ для разбиения всего спектра на независимые области. Нужно в первую очередь для оптимизации модели, потому что если не разделять спектр на области то подбор модели с ~80-100 линиями у которых есть несколько параметров, дает в итоге примерно 300-500 подбираемых параметров, что даже для самых быстрых методов оптимизации потребует если не часы, то дни на расчет. Разделение же на области поможет сократить вычисления до нескольких минут.
